# Privacy policy

Global IMAP Server Search does not collect, transmit, sell, or share telemetry or personal data with the developer or any third party.

## Data accessed

The extension accesses the following data only to provide its search features:

- Configured mail accounts and IMAP folder metadata, to select the account and folders to search.
- Message headers and server-side IMAP search results, to display matching messages.
- Thunderbird address books, to suggest addresses for the `from:` and `to:` filters.
- Thunderbird tags, to suggest values for the `tag:` filter.
- Extension-local storage, to remember the result limit and the Spam/Junk/Trash preference.

Search expressions are sent through Thunderbird only to the IMAP server configured by the user. Message bodies are searched by that server and are not uploaded to Sensnology or another service operated by the extension developer.

The extension does not include analytics, advertising, remote code, tracking pixels, or an external web service. It does not use Gloda or create a separate local full-text index.

## Data retention

The extension retains only its two user preferences in Thunderbird's local extension storage. Search results exist in memory while Thunderbird is running and are not sent to the developer.
